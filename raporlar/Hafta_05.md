# 5. Hafta Raporu: Kullanıcı Arayüzü Başlangıç (Frontend)

## Hedef
Arka planda (Backend) biriken yüksek hacimli skor ve puan verisini son kullanıcıların göz zevkine hitap edecek modern bir tasarımla sunabilmek. 

## Neler Yapıldı? 
Şu ana dek gerçekleştirilen altyapı işlemleri ekrana dökülmeye başlandı.
* **Arayüz Kodlaması:** ASP.NET Core MVC projelerinde (veya HTML/JavaScript entegrasyonuyla) Tailwind CSS kullanılarak modern ve şık duyarlı (responsive) bir arayüz tasarımına başlandı.
* **Grid ve Tablo Konfigürasyonu:** Lig bazlı puan durumu sayfasının şablon iskeleti çizildi. Bir tablo yapısı kurularak; oynanan maç (O), galibiyet (G), beraberlik (B), mağlubiyet (M), atılan-yenilen gol ve puan gibi spesifik metriklerin düzenli şemaları CSS sınıflarıyla belirlendi.
* **Veri Eksikliklerinin Aşılması:** UI sayfasında test edilen verilerin tam eşlenmesi ve "Veri Yok" görünümlerinin düzeltilmesiyle ilgili testler gerçekleştirildi. Backend API verisi tablolarla doğrudan ilişkilendiriliyor.
* **Hata Giderme ve Çekirdek Estetik:** Lig logoları veya sekmelerinin ekranda nasıl yerleşeceğine dair kararlar HTML ve CSS kodlarına uygulandı.

## Sonuç ve Çıktı
Web tarayıcısı üzerinden projeye girildiğinde estetik bir "Superliga" Puan durumu tablosuna erişmek mümkün. Veriler dinamik hale getiriliyor, entegrasyonun son adımları olan test süreci hâlihazırda projemizin mevcut aşamasını yansıtmaktadır.
